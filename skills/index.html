Get Started

Bitpanda Agent Skill

A pre-packaged agent skill for querying Bitpanda accounts. Drop it into any agent that follows the Agent Skills Open Standard — Claude Code, Cursor, custom orchestrators — and your assistant gains read-only access to portfolio, trades, prices, and transactions.

Read-only by design. Every command in this skill is read-only — no trading, orders, or write operations. Output is always JSON for clean agent parsing.

How it works

An agent skill is a directory with a SKILL.md manifest, executable scripts, and reference docs. The agent loads SKILL.md as context and learns when and how to invoke each command. The Bitpanda skill ships:

  • SKILL.md — agent-facing instructions and command catalog
  • scripts/bitpanda.sh — bash CLI that calls the Bitpanda API
  • references/api_reference.md — endpoint docs the agent can read

Authentication is by API key (BITPANDA_API_KEY environment variable). The skill is fully open-source, audited line-by-line.

Install

The skill is a plain directory you drop into your agent's skills folder. Requirements: curl, jq, and a Bitpanda API key.

  1. Clone the skill

    Place it wherever your agent expects skills (e.g. ~/.claude/skills/ for Claude Code).

    shell
    git clone https://github.com/bitpanda-labs/agent-skills.git
cp -r agent-skills/bitpanda ~/.claude/skills/
  2. Generate an API key

    Visit web.bitpanda.com/my-account/apikey and create a key with read-only scopes.

  3. Set the environment variable
    shell
    export BITPANDA_API_KEY="your-api-key-here"

    Add it to your shell rc file (~/.bashrc / ~/.zshrc) to persist across sessions.

  4. Verify
    shell
    bash bitpanda/scripts/bitpanda.sh balances --page-size 5

    Should print JSON with up to five wallet entries.

Commands

The skill exposes these commands to your agent. Every one returns JSON.

portfolio
portfolio

Aggregated holdings with EUR valuations across regular, staking, and index wallets.

trades
trades

Buy/sell history with asset name resolution, type classification, and date filtering.

price
price <SYMBOL>

Current price, currency, and daily change for a single asset.

prices
prices

Prices for all held assets, or every listed asset with --all.

tx
all-transactions

Full transaction log including rewards, deposits, withdrawals, and staking.

balances
balances

Raw wallet list with pagination and asset-type filtering.

asset
asset <asset_id>

Resolve an asset UUID to its name and symbol.

Manual invocation

You can also run the script directly — useful for debugging or shell scripts.

shell
bash bitpanda/scripts/bitpanda.sh portfolio
bash bitpanda/scripts/bitpanda.sh price BTC
bash bitpanda/scripts/bitpanda.sh trades --operation buy
bash bitpanda/scripts/bitpanda.sh all-transactions --page-size 100

Example prompts

Once installed, just ask your agent in plain language. It will pick the right command on its own.

"Show me my top five Bitpanda holdings by EUR value."
"How much BTC did I buy last quarter, and at what average price?"
"Sum my staking rewards from 2024 and break them down by asset."
"What's the live price of ETH and how has it moved today?"

FAQ

Is this safe? Can the agent move my money?

No. The skill only ever calls read-only Bitpanda API endpoints. There is no command for placing orders, transferring funds, or modifying account state. You can audit every script in the public repo before installing.

What's the difference between this and the MCP server?

The MCP server is a hosted service that AI clients (Claude, ChatGPT, Cursor) connect to over the Model Context Protocol with OAuth. The agent skill is a local script bundle that any agent following the Agent Skills standard can use directly — no server, no OAuth, just an API key in your environment.

Use MCP for consumer AI clients. Use the skill for local agents, custom orchestrators, or environments where you'd rather keep everything on your machine.

Which agents support this format?

Any tool that follows the Agent Skills Open Standard — Claude Code, Cursor, and a growing list of custom orchestrators. Since the skill is just a directory with a SKILL.md manifest, it's also trivial to wrap into your own framework.

Where do I get an API key?

From web.bitpanda.com/my-account/apikey. Generate a key with the read scopes you need (the skill only ever uses read scopes).
